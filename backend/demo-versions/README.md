# Demo Version Files

This directory contains hardcoded PDF files for demo version control.

## Structure

```
demo-versions/
├── resumes/
│   ├── resume-1-v1.pdf  (Version 1 of Resume ID 1)
│   ├── resume-1-v2.pdf  (Version 2 of Resume ID 1)
│   ├── resume-1-v3.pdf  (Version 3 of Resume ID 1)
│   ├── resume-2-v1.pdf  (Version 1 of Resume ID 2)
│   ├── resume-2-v2.pdf  (Version 2 of Resume ID 2)
│   └── resume-2-v3.pdf  (Version 3 of Resume ID 2)
└── cover-letters/
    ├── cover-letter-1-v1.pdf  (Version 1 of Cover Letter ID 1)
    ├── cover-letter-1-v2.pdf  (Version 2 of Cover Letter ID 1)
    ├── cover-letter-1-v3.pdf  (Version 3 of Cover Letter ID 1)
    ├── cover-letter-2-v1.pdf  (Version 1 of Cover Letter ID 2)
    ├── cover-letter-2-v2.pdf  (Version 2 of Cover Letter ID 2)
    └── cover-letter-2-v3.pdf  (Version 3 of Cover Letter ID 2)
```

## Instructions

1. Place your PDF files in the appropriate directories
2. Name them according to the pattern: `{type}-{id}-v{version}.pdf`
3. The system will automatically serve these files when viewing versions

## Example

For Resume ID 1:
- `resumes/resume-1-v1.pdf` - First version
- `resumes/resume-1-v2.pdf` - Second version  
- `resumes/resume-1-v3.pdf` - Third version

For Cover Letter ID 1:
- `cover-letters/cover-letter-1-v1.pdf` - First version
- `cover-letters/cover-letter-1-v2.pdf` - Second version
- `cover-letters/cover-letter-1-v3.pdf` - Third version
